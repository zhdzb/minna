# 《大家的日语》随身私教 App - 产品方案设计

基于您目前 N5 的日语水平，以及考级（JLPT）、锻炼听读能力、赴日工作的核心诉求，这套产品方案将完全对标《大家的日语》教材主干线，针对性解决“练习繁琐、无正向反馈、词汇卡壳”的痛点。

---

## 一、 产品核心定位与理念
- **定位**：基于《大家的日语》教材的“伴读式”智能练习平台。
- **理念**：**“抽离记忆负担，专注语法输出”**。在造句练习中直接提供核心生词提示，让用户专注句型结构；配合 AI 即时批阅，打造“做题-反馈-复习”的完整学习闭环。

---

## 二、 核心功能模块设计

### 1. 记忆导航与课程主线（进度管理）
- **书签记忆系统**：App 首页即是当前的进度看板。无论何时打开 App，系统会展示“上次练习到了：第 X 课 - Y 题型”，点击即可无缝衔接。
- **章节解锁模式**：按照《大家的日语》教材（1~50 课）顺序展开。在每一课下，整合该课涉及的**文型（句型）、例文、核心单词**，作为该课练习的生成边界。

### 2. 动态化分级练习体系
不再死板地看图造句。每一课的练习库将拆分为多种题型，覆盖您的词汇、语法、阅读与听力目标，并区分难度标签：
- **【Level 1: 基础语法固化】（选择/填空）**
  - **形式**：类似 JLPT N5/N4 考试中的单词、助词选择题。
  - **目标**：快速校验本课基本助词和时态变形。
- **【Level 2: 降维引导式造句】（翻译/造句）**
  - **痛点解决**：替代纸质书费时费力的练习 B/C。
  - **形式**：给出中文意思，要求输入日语翻译。
  - **单词提示卡 (Vocab Hint)**：在题目下方，**明确给出句中核心词汇的【中日对照+假名+罗马音】**。（例如：题目为“我明天去京都” 👉 提供词汇 `明日(あした)`、`京都(きょうと)`、`行きます(いきます)`）。用户只需思考**如何用本课语法把它们连接起来（时态、助词）**。
- **【Level 3: 听读实战训练】（听力理解/阅读）**
  - **听力**：播放一句结合本课语法的标准日音场景对话，用户需写出听到的核心句或选择对话意图。
  - **阅读**：系统用当前课及之前学过的语法，自动生成小短文，附带阅读理解题，锻炼长句解析能力。

### 3. AI 批改与智能讲评模块
所有题目（特别是主观造句题）在用户提交答案后，系统立刻进行秒级批阅：
- **正误判别与细节纠错**：高亮用户输入的错误部分（如助词用错 `に` 误用为 `で`，或者动词变形错误 `行きます` 误拼为 `行さます`）。
- **知识点讲解**：不仅给出正确答案，还会输出解析（“这里表示动作发生的目的地，应使用助词 に 或 へ”）。
- **自然度建议**：即使语法全对，AI 也会根据赴日工作需要的“地道日语”，给出更自然的表达建议。

### 4. 智能错题本与双轨收藏夹
为了更好地沉淀学习数据，产品设计了高度自定义的错题体系：
- **自动收录 (Auto-Mistake)**：凡是系统判定回答错误的题目，自动加入【错题本】并附带关联课文标签。
- **手动标记 (Manual Tagging)**：
  - 题目下方有两个显眼按钮：`[❗ 标记为错题/不会做]` 和 `[⭐ 收藏本题]`。
  - **适用场景**：用户即使做对了（可能是蒙对的或犹豫了很久），也能手动点 `[❗ 标记]` 归入错题；遇到非常实用的地道表达，可以点 `[⭐ 收藏]` 放进私人语料库。
- **复习模式**：支持按课数、按题型（如只重复练习造句错题）、按标签组（错题/收藏）进行随机抽取复习。

## 三、 用户操作流程示例（User Journey）

1. **进入 App** 👉 首页提示：上次学习到《大家的日文 1》第 6 课。
2. **选择练习** 👉 点击“造句演练（中等难度）”。
3. **开始答题** 👉 
   - 屏幕显示：“我用日语写信。”
   - 单词提示栏：`日本語（にほんご）`、`手紙（てがみ）`、`書きます（かきます）`。
   - 用户输入框：`わたしは 日本語 に 手紙を 書きます。`（故意写错助词）。
4. **提交反馈** 👉 
   - 扣分提示：❌ 助词错误。
   - 解析：“表示工具、手段时，应使用助词『で』，正确答案为『日本語 で 手紙を 書きます』”。
5. **归档处理** 👉 此题通过系统自动被加入错题本。用户觉得这句很实用，又顺手点了一下 `[⭐ 收藏]`。
6. **离开 App** 👉 进度自动保存至当前关卡。

## 五、 技术实现方案与跨设备同步设计

鉴于您希望**在不同设备间同步进度，又不想专门购买云服务器**，同时提到使用 **Google AI Studio (Gemini API)** 的需求，我们设计了这套**“纯静态网页 + 本地 JSON 数据驱动 + Git 同步”**的极简架构：

**架构设计：单一静态页面应用 (SPA) + Local JSON + Github 同步**

1. **前端与 AI 能力 (AI Studio 融合)**：
   - 界面使用纯 HTML/JS/CSS（或引入轻量级的 React/Vue 构建）。
   - 在代码中写入调用 **Google Gemini API** 的逻辑。您在 Google AI Studio 申请一个免费的 API Key，在 App 的设置页填入即可。AI 负责根据《大家的日语》范围出题、批改。

2. **数据存储：JSON 文件驱动 (`data.json`)**：
   - 设立一个统一的数据文件 `data.json`，包含了您的进度、错题本、收藏夹。
   - 每次您打开本地项目，App 自动读取这个 `data.json` 初始化状态。
   - 当您在应用中产生新数据（如做完第5课，新增了错题），App 会生成并**下载/覆盖项目根目录下的 `data.json`**（或者利用 Node.js 起一个几行代码的极简本地服务自动写回文件）。

3. **跨设备进度同步 (Github Workflow)**：
   - **准备工作**：将整个项目文件夹作为 Git 仓库推送到您的私有 Github 仓库。
   - **开始学习 (设备 A)**：在新设备上 `git pull` 拉取最新代码和最新的 `data.json`。双击运行，无缝衔接进度。
   - **学习结束 (设备 A)**：关闭前，最新学习产生的数据已保存更新在本地的 `data.json` 里。您通过一句脚本或命令行完成 `git add .` -> `git commit` -> `git push`。
   - **切换设备 (设备 B)**：再去公司或家里的设备上 `git pull` 一下，所有的错题和进度就完全同步过来了。

这套方案**极其极客、完全免费、隐私100%可控**，数据永远握在自己手里。不需要复杂的数据库搭建，只需懂基础的 Git 操作即可实现完美的跨端“云存档”。
